## План работы по созданию галереи изображений

**1. Анализ требований и планирование**

- Внимательно изучить техническое задание и макет Figma.
- Разбить проект на задачи (например, в ClickUp): верстка, стилизация, работа с API, модальное окно, поиск, валидация, деплой и т.д.
- Определить, какие изображения и данные понадобятся для тестирования (создать моковые данные на первом этапе).
- Настроить репозиторий Git, выбрать ветвление по GitFlow, описать правила Conventional Commits.

**2. Подготовка окружения**

- Создать структуру проекта: папки для HTML, CSS, JS, изображения (если есть локальные).
- Настроить конфиги форматирования кода (например, .editorconfig, Prettier для CSS).
- Подготовить базовый шаблон HTML с подключением стилей и скриптов.

**3. Верстка основных блоков**

- Разметить семантическую структуру страницы: header, main, section, nav, footer.
- Реализовать шапку с логотипом и навигацией (Gallery).
- Сделать якорные ссылки для плавной прокрутки до блока галереи.

**4. Стилизация и тема**

- Создать файл темы: переменные для цветов, отступов, шрифтов.
- Оформить базовые стили для адаптивности (медиа-запросы, flex/grid).
- Прописать правила именования классов, короткие и информативные имена.

**5. Реализация поисковой строки**

- Добавить блок с input и кнопкой поиска.
- Реализовать фильтрацию изображений по названию (без учёта регистра).
- Настроить валидацию: min/max длина, запретить спецсимволы, кроме разрешённых (использовать регулярные выражения[1]).

**6. Интеграция с API изображений**

- Зарегистрироваться на Unsplash, получить ACCESS_KEY (не коммитить в репозиторий!).
- Реализовать функцию загрузки изображений через JS (fetch).
- Отобразить изображения сеткой (flex/grid), добавить alt и подписи.

**7. Взаимодействие с пользователем**

- Реализовать выделение изображения при наведении (анимация, тень и т.д.).
- При клике — открыть модальное окно с увеличенным изображением.
- Добавить размытие и блокировку основной галереи при открытом модальном окне.

**8. Модальное окно и карусель**

- Реализовать модальное окно с возможностью листать изображения (влево/вправо).
- Реализовать цикличность перехода (последнее → первое и наоборот).
- Добавить закрытие по кнопке и клику вне области модального окна.
- Продумать анимацию появления/скрытия окна.

**9. Финальная стилизация и анимации**

- Добавить анимации для кнопок, появления элементов, наведения.
- Проверить адаптивность на разных устройствах и браузерах.

**10. Тестирование и рефакторинг**

- Провести ручное тестирование всех сценариев.
- Проверить валидацию, работу поиска, навигацию, модальное окно.
- Провести рефакторинг кода, убедиться в чистоте и читаемости.

**11. Деплой и документация**

- Задеплоить проект на выбранную платформу (GitHub Pages, Netlify, Vercel).
- Оформить README с инструкциями по запуску.
- Проверить, что ACCESS_KEY не попал в публичный репозиторий.

**12. Итоговая проверка**

- Сверить результат с макетом Figma.
- Проверить все требования ТЗ.
- Провести финальную ревизию кода и коммитов по Conventional Commits.

---

Этот план позволит структурировать работу, минимизировать ошибки и обеспечить соответствие всем требованиям тестового задания.
